# Recursive Parity

This file covers `test/proflog/recursive_synthesis_test.clj`.

The program is the simpler mutually recursive parity relation:

```clojure
even(x) :- x = zero
        or exists y. x = s(y) and odd(y)

odd(x)  :- exists z. x = s(z) and even(z)
```

## Higher Ground Success Cases

The committed current examples are:

```clojure
even(2) => succeeds
odd(3)  => succeeds
even(4) => succeeds
```

These show that the recursive branch still behaves correctly after the trivial
base case.

## Opposite-Parity Ground Failure Cases

The namespace also checks the negative side directly:

```clojure
odd(0)  => fails
even(1) => fails
odd(2)  => fails
even(3) => fails
```

Together with the success cases, this gives a small but useful semantic
cross-check of both mutually recursive clauses.

## Positive Witness Enumeration

The test suite keeps open-query synthesis on the positive side of the
semidecision boundary by probing a finite witness list directly.

Current even witnesses:

```clojure

```

Current odd witnesses:

```clojure

```

These are not generated by unrestricted open search. They are confirmed by
running the direct proof search separately on each candidate witness.

## Source To Kernel Descent

The parity program is Fitting P1 in the common descent described in
[Frontend To Kernel Descent](./frontend-to-kernel-descent.md):

```prolog
even(x) :- x = zero
        or exists y. (x = s(y) and odd(y)).

odd(x) :- forall y. (even(y) -> x != y).
```

The prefix frontend shape is:

```clojure
(pf/proflog p1-language
  (|- (even x)
      (or (= x zero)
          (exists [y]
            (and (= x (s y))
                 (odd y)))))

  (|- (odd x)
      (forall [y]
        (implies (even y)
                 (!= x y)))))
```

A ground check such as `even(4)` descends to a ground `pos` formula and then to
`kernel/prove-program` through `query/query-succeeds` or
`query/query-fails`. The witness enumeration section is deliberately different:
it does not ask `query-answers` to synthesize all parity answers. Instead, the
test supplies a finite candidate list and runs the same ground proof search on
each candidate. The exported-answer parameters are therefore absent; the only
budgets are proof count, fuel, and the finite witness list chosen by the test.
